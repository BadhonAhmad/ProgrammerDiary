# FastAPI Error Handling

Your API crashes with a 500 Internal Server Error. The user sees a generic "Something went wrong". You have no idea what broke. Good error handling means the client gets a clear message, and you get enough context to debug.

## Automatic Validation Errors

FastAPI returns detailed validation errors automatically when request data doesn't match your models.

```python
@app.post("/users")
def create_user(user: UserCreate):
    return user
```

```text
POST /users with body: {"name": 123, "age": "hello"}

Response (422 Unprocessable Entity):
{
  "detail": [
    {
      "type": "string_type",
      "loc": ["body", "name"],
      "msg": "Input should be a valid string",
      "input": 123
    },
    {
      "type": "int_parsing",
      "loc": ["body", "age"],
      "msg": "Input should be a valid integer",
      "input": "hello"
    }
  ]
}
```

No code needed — Pydantic + FastAPI handles it.

## HTTPException

For business logic errors, raise `HTTPException`:

```python
from fastapi import HTTPException, status

@router.get("/users/{user_id}")
def get_user(user_id: int, db: Session = Depends(get_db)):
    user = db.query(User).filter(User.id == user_id).first()
    if not user:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="User not found",
        )
    return user
```

```text
HTTPException:
  - Returns immediately (no code after it runs)
  - Client receives: {"detail": "User not found"} with status 404
  - Any exception in a route becomes an HTTP error response

Common status codes:
  400  Bad Request          → Invalid input from client
  401  Unauthorized         → Missing or invalid authentication
  403  Forbidden            → Authenticated but not allowed
  404  Not Found            → Resource doesn't exist
  409  Conflict             → Duplicate resource
  422  Validation Error     → Auto-generated by FastAPI
  500  Internal Server Error → Your code crashed (bug)
```

## Custom Exception Handlers

### Global Error Handler

```python
class NotFoundError(Exception):
    def __init__(self, name: str, id: int):
        self.name = name
        self.id = id

@app.exception_handler(NotFoundError)
async def not_found_handler(request: Request, exc: NotFoundError):
    return JSONResponse(
        status_code=404,
        content={
            "error": "not_found",
            "message": f"{exc.name} with id {exc.id} not found",
        },
    )
```

### Consistent Error Format

```python
class AppError(Exception):
    def __init__(self, status_code: int, error: str, message: str):
        self.status_code = status_code
        self.error = error
        self.message = message

@app.exception_handler(AppError)
async def app_error_handler(request: Request, exc: AppError):
    return JSONResponse(
        status_code=exc.status_code,
        content={
            "error": exc.error,
            "message": exc.message,
        },
    )

# Usage:
if not user:
    raise AppError(404, "not_found", "User not found")
if not authorized:
    raise AppError(403, "forbidden", "You cannot access this resource")
```

### Handle ALL Unhandled Exceptions

```python
@app.exception_handler(Exception)
async def global_exception_handler(request: Request, exc: Exception):
    logger.error(f"Unhandled exception: {exc}", exc_info=True)
    return JSONResponse(
        status_code=500,
        content={
            "error": "internal_error",
            "message": "An unexpected error occurred",
        },
    )
```

## Validation Error Customization

```python
from fastapi.exceptions import RequestValidationError

@app.exception_handler(RequestValidationError)
async def validation_handler(request: Request, exc: RequestValidationError):
    errors = []
    for error in exc.errors():
        errors.append({
            "field": ".".join(str(l) for l in error["loc"]),
            "message": error["msg"],
        })
    return JSONResponse(
        status_code=422,
        content={"error": "validation_error", "details": errors},
    )
```

## Key Points Cheat Sheet

| Concept | What It Does |
|---|---|
| **HTTPException** | Raise to return an error response immediately |
| **422 auto-generated** | Validation errors returned automatically by Pydantic |
| **Custom exception handler** | Global handler for specific exception types |
| **`@app.exception_handler`** | Register a handler for an exception class |
| **Consistent format** | Same error structure across all endpoints |
| **Global handler** | Catch-all for unhandled exceptions (logging + generic response) |
| **`RequestValidationError`** | Customize the default 422 validation error format |
| **`status` module** | Named constants for HTTP status codes |

**Good error handling is invisible when it works and obvious when it doesn't — return clear messages for expected errors, and log everything for the unexpected ones.**
